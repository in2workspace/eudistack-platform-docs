## Antes de abrir un ticket

2. Consulta el [changelog](reference/changelog.md) por si tu issue está relacionada con una versión reciente.

## Información que debes incluir

- **Tenant** afectado.
- **Componente**: Wallet, Issuer, Verifier, Portal, etc.
- **Pasos para reproducir** (lo más concretos posible).
- **Captura/log** del error (sanitiza datos personales).
- **Hora aproximada** del incidente y, si procede, `trace-id`.
